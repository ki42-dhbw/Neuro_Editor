🚀 Schnelleinstieg

1. Training (Lernen)

  1. Klicke oben auf 🤖 Simulator — das Spielfeld und die rechte Steuerleiste erscheinen.
  2. Optional: Rastergröße anpassen und NEU drücken.
  3. Drücke in der rechten Leiste auf 🚀 Turbo.
  4. Der Agent fällt anfangs oft in Löcher. Das ist normal!
  5. Wenn Pfeile auf dem Spielfeld einen konsistenten Weg zum Ziel zeigen → fertig gelernt.

2. Test-Modus (Prüfung)

  1. Aktiviere in der rechten Leiste: 🛡️ Test-Modus.
  2. Zufall (Epsilon) wird auf 0 gesetzt — nur reines Wissen.
  3. Drücke ▶️ Start.
  4. Der Agent sollte jetzt fehlerfrei zum Ziel laufen.

Was ist Reinforcement Learning?

Kernidee
Ein Agent lernt durch Versuch und Irrtum. Er bekommt Belohnungen (positiv) oder Bestrafungen (negativ) und optimiert sein Verhalten, um langfristig die höchste Gesamtbelohnung zu erzielen — ganz ohne vorprogrammierte Regeln.
Die 4 Elemente des RL
Agent 🤖 — handelt in der Umgebung
Umgebung 🗺️ — das Gitter mit Start, Ziel und Löchern
Zustand (State) — die aktuelle Feldnummer des Agenten
Aktion — Links / Unten / Rechts / Oben (L D R U)

Belohnungsschema

🕳️ In Loch gefallen−5Bestrafung für Fehler, Episode endet
👣 Schritt auf freiem Feld−0.1Effizienz fördern (kurze Wege)

📊 Die Q-Tabelle: Das Gedächtnis

Der Agent hat kein Gehirn — er hat eine Q-Tabelle. Q steht für „Quality" (Qualität). Sie beantwortet: „Wie gut ist Aktion A in Zustand S?"

Aufbau der Tabelle

SpalteBedeutungBeispiel
# (Zeile)Feldnummer auf der Karte (State).Feld 0 = oben links (Start)
L / D / R / UDie 4 Richtungen: Links, Unten, Rechts, Oben.Spalte R = Wert für „nach Rechts"
ZahlenwertErwartete zukünftige Gesamtbelohnung.8.50 = führt sicher zum Ziel

Farbcodierung in der Q-Tabelle

Roter Hintergrund — Bester Wert
In jeder Zeile wird der höchste Q-Wert (die beste Aktion für diesen Zustand) mit rotem Hintergrund hervorgehoben. Das zeigt sofort welche Richtung der Agent wählen würde.
Grüner Zeilenhintergrund — Optimaler Pfad
Die Zeilen, die auf dem aktuell besten Weg von Start zum Ziel liegen, werden grün markiert. Diese Pfad-Hervorhebung aktualisiert sich nach jedem Schritt und nach dem Turbo-Training.
Flash-Animation — Aktuelle Position
Wenn der Agent sich bewegt, blinkt die entsprechende Zeile kurz auf und scrollt automatisch ins Sichtfeld. So lässt sich live verfolgen was der Agent gerade „denkt".
Zahlenwerte
Grüne Zahlen = positive Werte (führen zum Ziel)
Rote Zahlen = negative Werte (gefährliche Richtungen)
Grau / 0 = noch unbekannt, kein Training dort stattgefunden

Visualisierung auf dem Spielfeld

Pfeile
Zeigen auf jedem Feld in die Richtung des höchsten Q-Werts. Wenn alle Pfeile einen konsistenten Weg zum Ziel zeigen, ist das Training abgeschlossen.
Heatmap-Farben
Freie Felder färben sich leicht grün (positive Q-Werte) oder rot (negative) — die Intensität entspricht der Stärke des Wertes. Weißbleibende Felder wurden noch nicht erkundet.

Lernfortschritt erkennen

  1. Anfang: Alle Werte = 0. Agent wandert zufällig. Kein Rot in der Tabelle.
  2. Erste Treffer: Zielfeld-Zeile bekommt positive Werte → erste rote Markierung.
  3. Propagation: Die Werte „wandern" rückwärts vom Ziel zum Start. Mehr Zeilen bekommen rote Markierungen.
  4. Fertig: Die grün markierten Pfad-Zeilen bilden eine lückenlose Kette von Start bis Ziel.

🧮 Die Bellman-Gleichung

Nach jedem Schritt aktualisiert der Agent genau einen Wert in der Q-Tabelle:

Q_neu = Q_alt + α · (r + γ · max Q_nächst − Q_alt)

Auf Deutsch erklärt

„Mein neues Wissen (Q_neu) = altes Wissen + ein bisschen Korrektur."

Die Korrektur ist der Unterschied zwischen dem was ich erwartet habe und dem was wirklich passiert ist.

Variablen

  • Q_alt — bisherige Schätzung
  • α (Alpha) — Lernrate
  • r — Sofort-Belohnung dieses Schritts
  • γ (Gamma) — Diskontfaktor: Gewicht der Zukunft
  • max Q_nächst — bester bekannter Wert des Folgefelds

Rechenbeispiel — erster Treffer am Ziel

Ausgangssituation
Agent steht auf Feld vor dem Ziel. Q_alt = 0. Reward = +10. Alpha = 0.7. Gamma = 0.95.
Q_neu = 0.0 + 0.7 · (10.0 + 0.95 · 0 − 0.0) = 7.0

Beim nächsten Mal weiß der Agent: „Dieser Schritt bringt 7 Punkte!" Die Information pflanzt sich rückwärts fort bis zum Start — sichtbar als wachsende rote Markierungen in der Q-Tabelle.

Temporal Difference (TD) Learning

Q-Learning ist ein model-freies TD-Verfahren — der Agent braucht kein Modell der Umgebung. Er lernt direkt aus Erfahrung, Schritt für Schritt, ohne auf eine vollständige Episode warten zu müssen.

🧠 Deep Reinforcement Learning (DQN)

Kernidee in einem Satz
Statt einer Tabelle mit einem Wert pro Feld und Aktion lernt ein neuronales Netz eine Funktion, die aus einer Zustandsbeschreibung direkt vier Q-Werte berechnet — einen für jede mögliche Richtung.

Q-Learning vs. Deep Q-Network (DQN)

Was geht ins Netz hinein? (Eingaben)

Die Zustandsbeschreibung
Das Netz bekommt zwei normalisierte Zahlen — die Zeilen- und Spaltennummer des Agenten, jeweils geteilt durch die maximale Anzahl. Steht der Agent auf Feld (Zeile 1, Spalte 2) in einem 4×4-Gitter:

row_norm = 1 ÷ (4−1) = 0.333
col_norm = 2 ÷ (4−1) = 0.667
Beide Werte liegen immer zwischen 0.0 (oben/links) und 1.0 (unten/rechts). Diese Normalisierung hilft dem Netz, alle Felder gleich gut zu verarbeiten — unabhängig von der Gittergröße.

Was kommt aus dem Netz heraus? (Die 4 Ausgabewerte)

Das Netz gibt gleichzeitig vier Zahlen zurück — einen Q-Wert für jede mögliche Richtung. Diese Zahlen sind nicht begrenzt (kein Softmax), können also positiv, negativ oder Null sein.

AusgangRichtungBedeutung des Wertes
Q(L) ← Links Geschätzte zukünftige Gesamtbelohnung, wenn der Agent von diesem Feld nach links geht
Q(D) ↓ Runter Geschätzte zukünftige Gesamtbelohnung, wenn der Agent nach unten geht
Q(R) → Rechts Geschätzte zukünftige Gesamtbelohnung, wenn der Agent nach rechts geht
Q(U) ↑ Hoch Geschätzte zukünftige Gesamtbelohnung, wenn der Agent nach oben geht
Konkretes Beispiel — Feld neben dem Ziel
Das Netz gibt aus: Q(L)=−1.2 · Q(D)=−0.8 · Q(R)=7.4 · Q(U)=0.3

→ Der Agent wählt Rechts, weil Q(R)=7.4 der höchste Wert ist.
→ Der Wert 7.4 bedeutet: „Wenn ich von hier nach rechts gehe, erwarte ich insgesamt ca. 7.4 Punkte Belohnung (inkl. Zukunft, diskontiert mit Gamma)."

Entscheidungsregel: Wähle immer die Aktion mit dem höchsten Q-Wert — argmax(Q(L), Q(D), Q(R), Q(U)).
Negative Werte
Negative Q-Werte bedeuten, dass diese Richtung mit hoher Wahrscheinlichkeit in ein Loch oder in eine Sackgasse führt. Je negativer der Wert, desto sicherer ist das Netz, dass diese Aktion schlecht ist. Werte nahe 0 bedeuten: das Netz hat dort noch wenig gelernt.

Schritt 1: Vorwärtsdurchlauf (Forward Pass)

Bevor der Agent handelt, berechnet das Netz alle vier Q-Werte für den aktuellen Zustand. Das passiert in zwei Rechenschritten:

Schicht 1 (Hidden):
hi = ReLU( b1[i] + W1[i][0]·row + W1[i][1]·col )    für i = 0 … 31

Schicht 2 (Ausgabe):
Q(a) = b2[a] + W2[a][0]·h0 + … + W2[a][31]·h31    für a ∈ {L, D, R, U}
  • W1 [32×2] — Gewichtsmatrix Eingabe→Hidden (64 Werte)
  • b1 [32] — Bias-Vektor der Hidden-Schicht (32 Werte)
  • ReLU(x) = max(0, x) — negative Voraktivierungen werden auf 0 gesetzt
  • W2 [4×32] — Gewichtsmatrix Hidden→Ausgabe (128 Werte)
  • b2 [4] — Bias-Vektor der Ausgabeschicht (4 Werte)

Insgesamt hat das Netz 196 trainierbare Parameter (64 + 32 + 128 + 4).

Schritt 2: Bellman-Zielwert berechnen

Nach der Aktion kennt der Agent das Ergebnis: Sofort-Belohnung r und Folgezustand s'. Daraus berechnet er den Zielwert, auf den das Netz trainiert wird:

target = r + γ · max( Q_Netz(s', L), Q_Netz(s', D), Q_Netz(s', R), Q_Netz(s', U) )

Wenn Ziel erreicht (r = +10)

Kein Folgezustand → max Q(s') = 0

target = 10 + 0.95 · 0 = 10.0

Der Ausgabeknoten der gewählten Aktion soll auf 10.0 trainiert werden.

Normaler Schritt (r = −0.1)

Bester Q-Wert im Folgezustand z. B. = 7.4

target = −0.1 + 0.95 · 7.4 = 6.93

Der Ausgangsknoten dieser Aktion soll auf 6.93 trainiert werden.

Schritt 3: Rückwärtsdurchlauf (Backpropagation)

Jetzt wird das Netz mit Gradientenabstieg angepasst. Nur der Ausgangsknoten der tatsächlich gewählten Aktion erhält einen Fehler — die anderen drei bleiben unverändert:

Fehler (nur für gewählte Aktion a):
δ = Q_Netz(s, a) − target

Verlust:
L = δ²    (Mean Squared Error)

Gewichtsänderung (Ausgabeschicht):
ΔW2[a][j] = − lr · δ · hj

Gewichtsänderung (Hidden-Schicht, nur wenn hi > 0):
ΔW1[i][k] = − lr · δ · W2[a][i] · xk
Warum nur eine Aktion trainieren?
Das Netz hat keinen Fehler für die nicht-gewählten Aktionen — der Agent weiß nicht, was dort passiert wäre. Nur für die tatsächlich ausgeführte Aktion liegt ein echtes Ergebnis vor. Die anderen drei Q-Werte bleiben bei dieser Trainingsrunde unverändert, auch wenn sie im Forward Pass berechnet wurden.
ReLU-Ableitung in der Hidden-Schicht
Beim Rückwärtsdurchlauf durch die ReLU-Aktivierung gilt:
dReLU/dx = 1 wenn h > 0, sonst 0

Neuronen, die im Forward Pass auf 0 lagen (weil ihre Voraktivierung negativ war), erhalten keinen Gradienten und werden in diesem Schritt nicht aktualisiert. Das ist die „tote Neuron"-Eigenschaft von ReLU.

Lernrate (lr)

Die Lernrate bestimmt, wie groß der Gradientenschritt bei jeder Trainingsrunde ist. Im DQN-Modus gibt es zwei Werte:

ModusLernrateWarum?
▶️ Schritt / Start0.005Konservativ — stabile Konvergenz Schritt für Schritt
🚀 Turbo (×100)0.01Schnellere Anpassung im Batch-Training, konvergiert in ~300–500 Episoden
zu groß (0.1+)—Netz überschwingt — Werte springen hin und her, kein stabiles Lernen
zu klein (0.0001)—Netz lernt extrem langsam, braucht tausende Episoden

Generalisierung — der Hauptvorteil

Was Q-Tabelle nicht kann, DQN schon
Die Q-Tabelle behandelt jedes Feld als völlig unabhängig. Das Netz hingegen lernt eine stetige Funktion über den Positionsraum. Felder, die geometrisch ähnlich liegen (z. B. beide weit oben rechts), erhalten automatisch ähnliche Q-Werte — auch wenn das Netz diese Felder nie direkt trainiert hat.

Beispiel: Das Netz lernt auf einem 4×4-Gitter, dass „nah am Ziel unten rechts" große positive Q-Werte für Bewegungen nach unten-rechts bedeutet. Dieses Wissen überträgt es automatisch auf andere Felder in ähnlicher relativer Lage.

Netzwerk-Visualisierung in der Steuerleiste

Im 🧠 Deep RL-Modus zeigt die rechte Steuerleiste eine Live-Visualisierung des Netzes:

  • Verbindungen: Grüne Linien = positive Gewichte, rote Linien = negative Gewichte. Die Helligkeit entspricht dem Betrag des Gewichts.
  • Eingabeneuronen (blau): Helligkeit zeigt den normalisierten Positionswert (0 = dunkel, 1 = hell).
  • Hidden-Neuronen (lila→gelb): Zeigen die ReLU-Aktivierung nach dem Forward Pass. Dunkel = inaktiv (0), hell/gelb = stark aktiv.
  • Ausgabeneuronen: Jeder der 4 Ausgangsknoten zeigt einen Richtungspfeil (←↓→↑), den Q-Wert mit Vorzeichen und den Richtungsnamen (Links/Runter/Rechts/Hoch). Der Knoten mit dem höchsten Q-Wert leuchtet gold — das ist die aktuell gewählte Aktion.
  • Die Anzeige aktualisiert sich bei jedem Schritt des Agenten.

Test-Modus im DQN

„Kein Zufall · Netz eingefroren"
Im DQN-Modus bewirkt der Test-Modus zwei Dinge gleichzeitig:

1. Kein Zufall (ε = 0) — der Agent handelt ausschließlich nach dem höchsten Q-Wert des Netzes, keine zufälligen Aktionen.
2. Netz eingefroren — die Gewichte W1/b1/W2/b2 werden nicht mehr aktualisiert. Der Agent läuft im reinen Inferenz-Modus — kein Backpropagation, kein Lernen mehr.

Das stellt sicher, dass der Test das tatsächlich Gelernte zeigt und das Training nicht verfälscht.

🎛️ Parameter-Einstellungen

Alle Schieberegler befinden sich in der rechten Steuerleiste unter „Einstellungen".

🎲 Epsilon — Entdeckerdrang (Exploration vs. Exploitation)
Das wichtigste Konzept! RL balanciert zwischen Exploration (Neues ausprobieren) und Exploitation (Bekanntes nutzen).

ε = 1.0 (100%): Vollständig zufällig. Ideal zu Beginn — alles erkunden.
ε = 0.0 (0%): Nur Wissen nutzen. Ideal am Ende — perfekt ausführen.

Strategie: Epsilon sinkt nach jedem Turbo-Durchlauf automatisch (Epsilon Decay × 0.99 pro Episode).
🎓 Alpha — Lernrate (nur Q-Learning)
Wie stark neue Informationen die alte Schätzung überschreiben.

α = 0.1: Sehr vorsichtig, braucht viele Episoden.
α = 0.7: Guter Standard-Wert für kleine Gitter.
α = 1.0: Jede neue Info überschreibt die alte komplett (instabil).

Im DQN-Modus hat Alpha keinen Einfluss — das Netz verwendet eine interne Lernrate (0.005 im Schritt/Start-Modus, 0.01 im Turbo-Modus), die direkt im Gradientenabstieg wirkt.
🔭 Gamma — Weitsicht (Diskontfaktor)
Wie wichtig ist die Zukunft im Vergleich zur Gegenwart?

γ = 0.0: Nur Sofort-Belohnung zählt. Kein Vorausdenken.
γ = 0.95: Agent denkt weit voraus. Standard für RL.
γ = 1.0: Fernste Zukunft genauso wichtig wie Gegenwart.
⏱️ Geschwindigkeit
Pause zwischen zwei automatischen Schritten (20–1000 ms). Nur beim ▶️ Start-Modus relevant. Turbo ignoriert die Geschwindigkeit und rechnet sofort.
⚡ Empfohlene Startwerte
Alpha 0.7 · Gamma 0.95 · Epsilon 1.0
→ Turbo mehrfach drücken bis Epsilon auf ~0.1 sinkt
→ Test-Modus aktivieren → ▶️ Start
