Neural Network Builder — Hilfe

Visueller Editor zum Erstellen, Simulieren und Trainieren neuronaler Netze

Der Neural Network Builder ermöglicht es, neuronale Netze vollständig grafisch aufzubauen: Neuronen per Drag & Drop platzieren, Verbindungen durch einfaches Klicken herstellen, Aktivierungsfunktionen zuweisen und das Netz entweder simulieren (Forward Pass) oder überwacht trainieren (Backpropagation).

Schnellstart 1. Neuronen aus der rechten Seitenleiste auf die Zeichenfläche ziehen  ·  2. Verbinden-Modus aktivieren und Neuronen verknüpfen  ·  3. Simulation starten oder Trainingsdaten eingeben
● Input-Neuron

Eingabe­größe des Netzes. Kein Aktivierungs­aufwand nötig.

● Hidden-Neuron

Verarbeitung in versteckten Schichten mit Aktivierungsfunktion.

● Output-Neuron

Ergebnis­knoten. Aktivierungs­funktion und Bias optional zuweisbar (für Klassifikation z. B. Sigmoid oder Softmax).

● Bias-Neuron

Konstanter Eingang = 1. Gewicht entspricht dem Bias-Wert.

Aktivierungsfunktionen

Aktivierungsfunktionen werden durch Drag & Drop aus der Seitenleiste auf ein Neuron gezogen. Alternativ: Rechtsklick auf Neuron oder Properties-Panel. Anwendbar auf Hidden- und Output-Neuronen (am Output standardmäßig deaktiviert; für Klassifikation explizit Sigmoid/Softmax wählen). Bias- und Input-Neuronen haben keine Aktivierung.

ReLU

Standard für Hidden Layers. Löst das Vanishing-Gradient-Problem.

f(x) = max(0, x)
Sigmoid

Ausgabe in [0, 1]. Gut für binäre Klassifikation. Neigt zu Vanishing Gradient.

f(x) = 1 / (1 + e⁻ˣ)
Tanh

Ausgabe in [−1, 1]. Zentriert um 0 — besser als Sigmoid für Hidden Layers.

f(x) = tanh(x)
Softmax

Normalisiert mehrere Ausgaben zu Wahrscheinlichkeiten (Summe = 1). Mehrklassen­klassifikation.

f(xᵢ) = eˣⁱ / Σeˣʲ
Linear

Keine Nichtlinearität. Für Regression oder als Pass-Through-Schicht.

f(x) = x
Leaky ReLU

Behebt das „Dead Neuron"-Problem von ReLU durch kleinen negativen Gradienten.

f(x) = x ≥ 0 ? x : αx (α ≈ 0.15)

Aktivieren

  2. Eingabewerte in den Input-Feldern auf dem Canvas oder im Seitenpanel setzen.

  • Input(shape=(N,)) → N Input-Neuronen.
  • Erste bis vorletzte Dense-Schicht → Hidden-Neuronen mit der angegebenen Aktivierung.
  • Letzte Dense-Schicht → Output-Neuronen mit der angegebenen Aktivierung (z. B. softmax für Mehrklassen-Klassifikation).

Was nicht unterstützt wird Keine Verarbeitung von Conv2D, LSTM, Dropout, mehreren Eingängen oder verzweigten Topologien. Solche Layer werden ignoriert. Nur sequenzielle Dense-Netze werden importiert.

Semantische Positionierung im 3D-Raum

Wenn Dokumente in den 3D-Raum importiert werden, bestimmt der Connector die Position nach dieser Priorität:

  1. Vokabular-Lookup: Ist das Wort im semantischen Wörterbuch (z.B. „Elefant", „Auto"), landet es exakt im richtigen Cluster.
  2. Tokenizer-Fallback: Bei kurzen unbekannten Texten wird der erste echte Token (kein Special Token, kein Subword) als Positions-Seed verwendet.
  3. Text-Hash: Bei langen Dokumenten (viele Tokens) wird ein stabiler Hash des Textes als Seed verwendet, damit Punkte gleichmäßig im Raum verteilt sind statt am Ursprung zu clustern.
Warum clustern lange Dokumente am Ursprung? Werden alle Token-Vektoren eines langen Textes gemittelt, konvergiert das Ergebnis gegen den Nullpunkt — je mehr Tokens, desto stärker (Gesetz der großen Zahlen: Durchschnittslänge ∝ 1/√n). Der Text-Hash-Fallback umgeht dieses Problem durch eine positionsunabhängige Platzierung.

Die Q-Tabelle zeigt für jeden Zustand (Gitterfeld) die gelernten Q-Werte für alle vier Aktionen:

SpalteAktion

Der höchste Q-Wert pro Zeile ist farbig hervorgehoben (beste Aktion). Die aktive Zeile (aktueller Agenten-Zustand) und der optimale Pfad werden ebenfalls markiert.

Was ist ein Q-Wert? Q(s, a) ist der erwartete kumulierte zukünftige Reward, wenn der Agent im Zustand s die Aktion a ausführt und danach optimal handelt. Die Bellman-Gleichung definiert das Update:
Q(s,a) ← Q(s,a) + α · [r + γ · max Q(s',a') − Q(s,a)]

Theoretischer Hintergrund

Bellman-Gleichung

Q-Update (Bellman-Gleichung) Q(s,a) ← Q(s,a) + α · [r + γ · max_a' Q(s',a') − Q(s,a)]

s = aktueller Zustand  ·  a = gewählte Aktion  ·  r = erhaltener Reward  ·  s' = Folgezustand  ·  α = Lernrate  ·  γ = Diskontierung

StrategieBedingungVerhalten
ExplorationZufallszahl < εZufällige Aktion wählen → neue Zustände erkunden
ExploitationZufallszahl ≥ εAktion mit höchstem Q-Wert wählen → bekanntes Wissen nutzen

Deep Q-Network (DQN)

Statt einer Tabelle approximiert ein neuronales Netz die Q-Funktion: Q(s, a; θ) mit Gewichten θ. Das Netz nimmt den Zustandsvektor als Eingabe und gibt Q-Werte für alle Aktionen aus. Trainiert wird mit MSE-Loss zwischen Ziel-Q-Wert (Bellman-Ziel) und Netz-Ausgabe.

Wann DQN statt Q-Tabelle? Bei großen oder kontinuierlichen Zustandsräumen würde eine Q-Tabelle zu groß. Ein neuronales Netz generalisiert — es kann Q-Werte auch für noch nie gesehene Zustände schätzen.

CNN-Pipeline: Was nach der Faltung passiert

1. Convolution (Faltung)

Kernel gleitet über das Bild → Feature Map mit Aktivierungsstärken pro Position.

P(x,y) = Σ I(x+i,y+j)×K(i,j) / Div + Bias
2. ReLU (Aktivierung)

Alle negativen Werte werden auf 0 gesetzt. Nur positive Erkennungen (Muster gefunden) passieren.

f(x) = max(0, x)
3. Max Pooling

2×2-Fenster gleitet über die Feature Map und behält nur den Maximalwert. Halbiert die räumliche Auflösung.

out = max(patch 2×2)

Effizienz: Toeplitz-Matrix statt Schleifen

In der Praxis wird die Faltungsoperation nicht mit vier verschachtelten Schleifen implementiert, sondern als eine einzige Matrixmultiplikation formuliert:

Vektorisierung (Toeplitz-Trick) Das 2D-Bild wird zu einem 1D-Vektor gestreckt (Flattening). Der Kernel wird zu einer großen Toeplitz-Matrix aufgebläht, deren Einträge entlang der Diagonalen periodisch wiederholt werden. Dann:

[Toeplitz-Matrix] × [Bildvektor] = [Feature-Map-Vektor]

Ein einziger hochgradig paralleler Rechenschritt auf der GPU — statt Millionen sequenzieller Schleifeniterationen.
